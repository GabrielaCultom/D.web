Software De Versionamento

O objetivo é realizar o controle de versões de um código ou documento. Sendo possivel gerenciar as versões de um código ou de documentos, realizados por desenvolvedores e suas equipes.

Controle de Versões

A maneira correta é por um software, que cria um repositório local, e apartir do seu arquivo, cria um histórico de versões.Podendo fazer tambem um repositório remoto, que tem seu servidor especifico.

Tipos de Software de Controle de versões

Centralizado
É o qual possui um repositório central em um servidor com controle de versões,e a atualização depende do servidor.
Distribuido
Possui um repositório local com controle de versões, e a atualização no repositório central, não depende do servidor.

GIT

Git (prevençao fe sistema global de rastreamento que surgiu em 2005, foi projetado e desenvolvido por Linus Torvaldes, a sua criaçao foi para o desenvolvimento do Kranel do lunex. O git é um repertorio com um historio completo e habilidade total das revisoes, não depende de um acesso a uma rede ou a um servidor central.

GITHUB

É a plataforma de hospedagem de codigos-fonte e arquivos com controle de versão.
Surgiu em 2008 mas só foi adquirido em 2018 pela Microssoft.Ele ospeda:

  • Mais de 100 milhões de projetos.
  • Mais de 56 milhões de usuarios.
Comandos do Git
COMMIT
Torna permanente um conjunto de alteraçoes em um repositorio.
PUSH
Envia alterações para o servidor no repositor remoto.
FETCH
Verifica se o seu repositor local esta em sicronia com o remoto.
PULL
Sincroniza o repositorio local com o mesmo remoto atualizando as versoes.

OBS: É sempre importante a toda atualizaçao do arquivo ou projeto, realizar o commit, tornando as atualizaçoes permanentes e em seguida realizar o push, que envia as alteraçoes para a plataforma.
